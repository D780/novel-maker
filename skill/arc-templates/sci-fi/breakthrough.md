# 科技突破弧

> 主角在科研瓶颈中挣扎，经历灵感闪现和实验验证，最终实现改变世界的科技突破

## 弧线结构

瓶颈 → 灵感 → 实验 → 突破 → 应用 → 影响

## 阶段定义

### 阶段 1：瓶颈（2-3章）
**目标：** 展示科研困境，建立突破的必要性
**关键事件：**
- 项目陷入停滞，经费或资源即将耗尽
- 同行质疑、团队士气低落
- 主角承受巨大压力，但拒绝放弃
**情绪走向：** 焦虑 → 压抑
**章节建议：** 2-3章

### 阶段 2：灵感（1章）
**目标：** 意外触发突破性想法
**关键事件：**
- 一个偶然事件（生活观察/跨学科启发/意外事故）点亮思路
- 主角意识到现有理论的盲点
- 产生大胆假设，开始重新设计方案
**情绪走向：** 惊喜 → 兴奋
**章节建议：** 1章

### 阶段 3：实验（2-3章）
**目标：** 验证假设，经历多次失败
**关键事件：**
- 设计实验方案，组建核心团队
- 前几次实验失败，数据不符预期
- 调整参数、改进设备，逐步逼近真相
**情绪走向：** 期待 → 紧张
**章节建议：** 2-3章

### 阶段 4：突破（1-2章）
**目标：** 关键实验成功，实现技术突破
**关键事件：**
- 最终实验成功，数据完美验证假设
- 团队震惊，主角确认突破的意义
- 发表论文或申请专利，引发学术界震动
**情绪走向：** 震撼 → 狂喜
**章节建议：** 1-2章

### 阶段 5：应用（2-3章）
**目标：** 将实验室成果转化为实际应用
**关键事件：**
- 产业化过程中的技术难题
- 商业竞争、资本介入、利益博弈
- 首个成功应用场景落地
**情绪走向：** 热血 → 成就感
**章节建议：** 2-3章

### 阶段 6：影响（1-2章）
**目标：** 展示科技对社会的深远影响
**关键事件：**
- 技术普及改变行业格局
- 伦理争议或社会讨论
- 主角回顾历程，展望未来
**情绪走向：** 感慨 → 希望
**章节建议：** 1-2章

## 关键场景

1. **深夜实验室**：主角独自面对失败的数据，突然灵光一闪的瞬间
2. **突破时刻**：实验成功的那一刻，数据曲线完美吻合，团队欢呼
3. **发布会/论文发表**：向世界展示成果，回应所有质疑

## 注意事项

- **避免神化**：突破不应是主角一人之功，需要团队协作
- **科学合理性**：虽然是小说，但突破的逻辑链条要自洽
- **节奏把控**：实验阶段不要过于冗长，用细节展现专业性而非堆砌术语
- **情感铺垫**：低谷期要足够压抑，突破时的爽感才足够强烈
