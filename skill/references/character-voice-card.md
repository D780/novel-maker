# 角色语音卡指南

> **设计参考文档** — 定义"如何设计语音卡"。检查规则详见 [rules/character-voice.md](../rules/character-voice.md)。

> 为每个角色设计独特的说话方式，确保对话区分度

## 什么是角色语音卡

角色语音卡是定义角色语言特征的文档，确保遮住角色名字也能认出是谁在说话。

## 核心原则

**一句话标准：** 遮住名字，能认出是谁说的

## 语音卡要素

### 1. 口头禅

角色高频使用的特定词语或短句。

**示例：**
```
角色A："有意思..."（每次遇到有趣的事）
角色B："本座..."（自称，显示身份）
角色C："算了算了"（口头禅，显示性格）
```

**设计要点：**
- 2-3个口头禅即可，不宜过多
- 要符合角色身份和性格
- 避免与其他角色重复

### 2. 常用词汇

角色偏好使用的词汇类型。

**示例：**
```
文雅角色：使用"此物"、"甚好"、"不妥"
粗犷角色：使用"这玩意儿"、"痛快"、"干他娘"
学者角色：使用"据我所知"、"从理论上看"
```

### 3. 禁用词汇

角色不会使用的词汇。

**示例：**
```
古代角色：禁用"OK"、"没问题"、"搞定"
粗人角色：禁用"或许"、"大概"、"理论上"
小孩角色：禁用"综上所述"、"鉴于此"
```

### 4. 语气特点

| 维度 | 选项 | 说明 |
|------|------|------|
| 语速 | 快/中/慢 | 影响句式长短 |
| 音量 | 大/中/小 | 影响感叹号使用 |
| 语调 | 平稳/起伏/尖锐/低沉 | 影响情感表达 |
| 句式 | 长句/短句/混合 | 影响表达风格 |

### 5. 特殊表达

角色独有的表达方式。

**示例：**
```
角色A：喜欢用反问句
角色B：喜欢用比喻
角色C：喜欢用排比
角色D：说话总是留半句
```

## 区分度检查

### 测试方法

**盲测法：** 遮住角色名字，看能否认出说话者

```
❌ 无法区分：
A："我觉得不行。"
B："我觉得不行。"
（两人说话方式完全一样）

✅ 可以区分：
A："这事...不妥。"（文雅，用省略号）
B："干他娘的，不行！"（粗犷，带脏字）
C："从理论上分析，成功率不足三成。"（学者，数据化）
```

### 区分度维度

| 维度 | 检查点 |
|------|--------|
| 自称 | 我/本座/老子/小的/俺 |
| 称呼 | 你/阁下/小子/大人 |
| 语气词 | 啊/呢/吧/嘛/哦 |
| 句式 | 长句/短句/疑问句/感叹句 |
| 词汇 | 文雅/粗俗/专业/口语 |

## 按角色类型设计

### 主角

- 口头禅体现核心性格
- 语言风格随成长变化
- 保持辨识度

### 配角

- 2-3个鲜明特征即可
- 避免与主角雷同
- 可用方言或职业用语

### 反派

- 语言体现动机和性格
- 可有独特的自称
- 避免脸谱化

### 女性角色

- 避免所有女性角色说话一样
- 性格差异体现在语言上
- 可使用不同的语气词

## 常见问题

### Q1：角色太多，语音卡管理不过来？
**A：** 优先为主要角色（主角+3-5个核心配角）设计语音卡，次要角色抓1-2个特征即可。

### Q2：角色成长后语言风格变化怎么办？
**A：** 在语音卡中标注"成长前/成长后"的变化，保持核心特征不变。

### Q3：如何避免角色说话像AI？
**A：** 参考 [反AI表达规则](../rules/anti-ai-expressions.md)，特别是"角色对话区分度"部分。

## 与其他系统的关系

| 系统 | 关系 |
|------|------|
| 角色档案 | 语音卡是角色档案的语言维度扩展 |
| 文风系统 | 语音卡在整体文风下体现角色个性 |
| 审计系统 | 审计时检查对话是否符合语音卡 |
