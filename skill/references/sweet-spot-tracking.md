# 爽点密度追踪

> 追踪和管理作品中的爽点分布，确保商业可读性

## 什么是爽点

爽点是让读者产生"爽快感"的情节节点，是网文商业性的核心要素。

## 爽点分类

### 按类型分

| 类型 | 说明 | 示例 | 适用场景 |
|------|------|------|---------|
| 打脸爽 | 嘲讽→反杀 | "你说我是废物？看好了" | 人际冲突 |
| 突破爽 | 升级→碾压 | 突破境界，一击秒杀 | 修炼/成长 |
| 逆袭爽 | 绝境→翻盘 | 必死之局，绝地反击 | 危机场景 |
| 装逼爽 | 淡然→震惊 | 随手一击，众人震惊 | 实力展示 |
| 复仇爽 | 血债→血偿 | 灭门之仇，一朝得报 | 恩怨清算 |
| 智谋爽 | 布局→收网 | 运筹帷幄，决胜千里 | 权谋/推理 |
| 情感爽 | 误会→和解 | 冰释前嫌，重归于好 | 感情线 |
| 身份爽 | 隐藏→揭示 | 乞丐竟是皇子 | 身份反转 |
| 收获爽 | 付出→回报 | 历经艰险，获得宝物 | 探险/任务 |
| 碾压爽 | 对手→秒杀 | 降维打击，毫无悬念 | 实力展示 |

### 按强度分

| 强度 | 说明 | 持续时间 | 使用频率 |
|------|------|---------|---------|
| S5 超级爽 | 全书级别高潮 | 3-5章铺垫 | 每卷1-2次 |
| S4 大爽 | 卷级别高潮 | 2-3章铺垫 | 每5-8章1次 |
| S3 中爽 | 章节级别高潮 | 1章铺垫 | 每3-5章1次 |
| S2 小爽 | 段落级别快感 | 半章铺垫 | 每1-3章1次 |
| S1 微爽 | 细节级别满足 | 即时 | 每章可有 |

## 爽点密度标准

### 按平台分

| 平台 | 最低密度 | 推荐密度 | 说明 |
|------|---------|---------|------|
| 番茄小说 | 每3章1个S2+ | 每2章1个S2+ | 免费阅读，需高密度 |
| 起点中文网 | 每5章1个S2+ | 每3章1个S2+ | 付费阅读，可适度铺垫 |
| 晋江文学城 | 每5章1个情感爽 | 每3章1个情感爽 | 女性向，情感爽为主 |

### 按位置分

| 位置 | 密度要求 | 说明 |
|------|---------|------|
| 前3章 | 至少1个S3+ | 开篇必须抓住读者 |
| 前10章 | 至少2个S3+ | 建立核心吸引力 |
| 每卷开头 | 1个S3+ | 新卷开篇需吸引 |
| 每卷结尾 | 1个S4+ | 卷末高潮 |
| 每5章 | 至少1个S3+ | 维持追读动力 |

## 追踪系统

### 爽点记录表

每章写完后，审计师记录爽点信息：

```json
{
  "chapter": 15,
  "sweet_spots": [
    {
      "type": "打脸爽",
      "intensity": "S3",
      "position": "章末",
      "description": "主角当众揭穿反派阴谋",
      "effectiveness": "A"
    }
  ],
  "total_intensity": "S3",
  "gap_since_last": 2
}
```

### 密度报告

每5章生成一次爽点密度报告：

```markdown
# 爽点密度报告（第11-15章）

## 统计
- 总爽点数：4
- S5: 0 | S4: 1 | S3: 2 | S2: 1
- 平均密度：每1.25章1个S2+
- 最大间隔：2章

## 分布图
```
Ch11: S2 (小爽)
Ch12: -- (无爽点)
Ch13: S3 (中爽 - 打脸)
Ch14: S3 (中爽 - 突破)
Ch15: S4 (大爽 - 逆袭)
```

## 评估
- 密度：✅ 达标（推荐每3章1个S2+）
- 分布：✅ 均匀
- 类型：⚠️ 缺少情感爽，建议增加
- 强度：✅ 有递进

## 建议
1. 第12章可增加一个小爽点
2. 后续可增加情感爽类型
```

## 爽点设计原则

### 1. 铺垫与释放

```
铺垫（压）→ 释放（爆）

铺垫越长，释放越爽：
- 1章铺垫 → S2 小爽
- 3章铺垫 → S3 中爽
- 5章铺垫 → S4 大爽
- 10章铺垫 → S5 超级爽
```

### 2. 类型多样化

避免连续使用同一类型爽点：
```
❌ 打脸→打脸→打脸→打脸（单调）
✅ 打脸→突破→智谋→复仇（多样）
```

### 3. 强度递进

爽点强度应随剧情递进：
```
第1卷：S2-S3 为主
第2卷：S3-S4 为主
第3卷：S4-S5 为主
```

### 4. 与情绪曲线配合

```
情绪单元：压-小扬-压-爆
对应爽点：S1-S2-S1-S3+

"压"时铺垫爽点
"爆"时释放爽点
```

## 常见问题

### Q1：爽点太少怎么办？
**A：** 检查是否有可加速的铺垫，将部分铺垫压缩，提前释放爽点。

### Q2：爽点太多读者疲劳怎么办？
**A：** 在高强度爽点后安排"缓冲章"（S1-S2），让读者情绪回落。

### Q3：如何设计"大爽"？
**A：** 大爽需要长铺垫（5章+），多线收束，让读者期待值拉满后再释放。

### Q4：情感爽怎么设计？
**A：** 先建立角色魅力，制造误会/分离，再安排和解/重逢。

## 爽点检查清单

每5章审查时检查：
- [ ] 是否有至少1个S3+爽点？
- [ ] 爽点类型是否多样（不重复）？
- [ ] 爽点间隔是否合理（不超过3章）？
- [ ] 爽点强度是否有递进？
- [ ] 爽点是否有铺垫（非突兀）？
- [ ] 是否符合平台密度要求？
