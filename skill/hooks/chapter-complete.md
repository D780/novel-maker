# Chapter Complete Hook

## 概述

章节完成后的自动化更新流程。由复盘师 sub-agent 执行，协调者通过 Task 工具调度。

复盘师读取最终稿件后，按照以下规则更新所有 truth-files。每个 truth-file 的更新应参考对应的模板文件，使用模板定义的结构。

## 触发方式

由协调者调度复盘师 sub-agent 时触发。复盘师读取最终稿件后执行本 hook 的所有检查。

## 模板索引

每个 truth-file 对应的模板文件定义了其完整结构：

| Truth-File | 模板文件 | 用途 |
|-----------|---------|------|
| `truth-files/current-state.md` | 无 | 当前状态快照（非模板化） |
| `truth-files/pending-hooks.md` | `skill/templates/hook-template.md` | 伏笔管理 |
| `truth-files/characters.md` | `skill/templates/character-profile.md` | 角色档案 |
| `truth-files/world-setting.md` | `skill/templates/world-setting.md` | 世界观设定 |
| `truth-files/power-system.md` | `skill/templates/power-system.md` | 力量体系 |
| `truth-files/emotional-arcs.md` | `skill/templates/emotional-arcs.md` | 情感弧线 |
| `truth-files/timeline.md` | `skill/templates/timeline.md` | 时间线 |
| `truth-files/subplot-board.md` | `skill/templates/subplot-board.md` | 支线看板 |

## 各 Truth-File 详细更新规则

### 1. `current-state.md`（每章必更）

更新内容：
- **世界状态**：当前所在地点、时间、天气/季节
- **角色状态**：每个出场角色的体力/灵力/情绪状态
- **剧情进展**：本章完成的核心事件
- **新获得的物品/技能**：本章获得的物品、技能、信息

### 2. `pending-hooks.md`（每章必更）

参考 `skill/templates/hook-template.md` 的结构：

**新伏笔埋设**：
```markdown
- 编号：H{next_id}
- 名称：[伏笔名称]
- 设置章节：ch{XXX}
- 描述：[伏笔详情]
- 暗示方式：对话暗示/行为暗示/环境暗示/物品暗示
- 预期回收章节：ch{YYY}
- 关联角色：[角色名]
- 关联伏笔：[伏笔编号]
```

**伏笔回收标记**：
- 将对应伏笔标记为"已回收"
- 记录实际回收章节和效果

**状态更新**：
- 调整未回收伏笔的预期回收章节
- 更新关联角色/伏笔信息

### 3. `characters.md`（条件更新）

当有**新角色出现**或**角色状态发生重大变化**时更新。参考 `skill/templates/character-profile.md` 的结构：

```markdown
## [角色名]

### 基本信息
- 身份/职业：[当前身份]
- 所属势力：[当前势力]

### 当前状态
- 等级/境界：[当前等级]
- 能力：[新增/变化的能力]
- 持有物品：[新增/变化的物品]

### 性格核心（仅初始化时填写）
- 核心价值观：[一话概括]
- 致命缺陷：[一话概括]
- 内心渴望：[一话概括]

### 角色关系变化
- [角色A]：[关系变化]
- [角色B]：[关系变化]

### 出场记录
| 章节 | 作用 | 高光时刻 |
|------|------|---------|
| ch{XXX} | [作用] | [描述] |
```

**注意**：首次出场时创建完整档案（基本信息 + 外貌描写 + 性格核心 + 背景故事），后续只更新状态变化。

### 4. `world-setting.md`（条件更新）

当有**新地点、新势力、新规则**出现时更新。参考 `skill/templates/world-setting.md` 的结构：

```markdown
### 新增地点
| 地点名 | 类型 | 位置 | 所属势力 | 说明 |
|--------|------|------|---------|------|

### 新增势力
| 势力名 | 类型 | 实力 | 地位 | 与其他势力关系 |
|--------|------|------|------|---------------|

### 新增规则/设定
- [规则描述]
```

### 5. `power-system.md`（条件更新）

当力量体系有**新等级、新技能、新法宝**出现时更新。参考 `skill/templates/power-system.md` 的结构：

```markdown
### 新增等级/突破
- [角色]突破至[等级]
- 等级表现：[特征描述]

### 新增技能
| 技能名 | 类型 | 等级 | 拥有者 | 描述 |
|--------|------|------|--------|------|

### 新增物品/法宝
| 物品名 | 类型 | 品级 | 拥有者 | 功能 |
|--------|------|------|--------|------|
```

### 6. `emotional-arcs.md`（条件更新）

当角色**情感状态发生转折**时更新。参考 `skill/templates/emotional-arcs.md` 的结构：

```markdown
### [角色名]
| 章节 | 情感状态 | 触发事件 | 强度 |
|------|---------|---------|------|
| ch{XXX} | [情感] | [触发事件] | X/10 |
```

**跨5章时**：检查情感弧线连续性，标记断裂点。

### 7. `timeline.md`（条件更新）

当**时间推进**或**跨5章**时更新。参考 `skill/templates/timeline.md` 的结构：

```markdown
| 章节 | 故事内时间 | 事件 | 地点 | 涉及角色 |
|------|----------|------|------|---------|
| ch{XXX} | [时间] | [事件] | [地点] | [角色] |
```

**时间冲突检测**：检查同一时间角色是否出现在两个地点。

### 8. `subplot-board.md`（条件更新）

当**跨幕**或**支线状态变化**时更新。参考 `skill/templates/subplot-board.md` 的结构：

```markdown
### [支线名]
- 状态：活跃/暂停/已收束
- 当前章节：ch{XXX}
- 进展：[更新进展]
- 预计收束：ch{YYY}
```

### 9. 系统/金手指角色追踪（条件更新）

当章节中出现**系统、金手指、有意识的神器/灵体、传承印记、血脉觉醒**等特殊存在时，需在 `characters.md` 中使用系统/金手指档案模板记录。

**检测信号**：
- 出现系统提示音/面板（"叮！"、"【系统提示】"）
- 主角获得外挂/金手指
- 有意识的神器/灵体与主角对话
- 传承印记/血脉觉醒触发
- 存在"系统任务"、"成就系统"等机制

**更新规则**：
1. **首次出现**：在 `characters.md` 中使用 `skill/templates/character-profile.md` 中的"系统/金手指档案"部分创建完整记录
2. **系统升级/功能解锁**：更新"系统等级/成长体系"和"成长记录"
3. **系统规则变化**：更新"系统规则/限制"
4. **系统人格变化**：更新"系统人格/意识"中的关系演变
5. **系统与宿主关系变化**：更新"系统人格/意识"中的关系阶段

**系统角色在 `characters.md` 中的标记方式**：
```markdown
## [系统名称]
- **类型**：系统/金手指
- **绑定宿主**：[宿主名]
- **当前等级**：Lv.X（详见下方系统档案）
```

**系统等级变化同时更新 `power-system.md`**：如果系统解锁了新的能力/技能，需同步更新到力量体系表中。

## 数据流说明

```
写手写入 temp/  →  审计师审查 temp/  →  复盘师归档到 novel/
    ↓                    ↓                      ↓
.novel-maker/temp/  审计报告保留         novels/volume-XX/chapters/
```

**关键规则**：
- 写手**只写入 temp/** 目录，不直接写入 novel/ 目录
- 审计师**只审查 temp/** 目录下的草稿
- 复盘师**负责将定稿从 temp/ 归档到 novel/** 目录
- 归档后**清理 temp/ 中的 draft.md 和 revised.md**，保留 audit.json

## 强制检查清单（复盘师执行）

### 1. Truth-Files 更新（必做）

**每章必更**：
- [ ] `current-state.md` — 世界状态 + 角色状态 + 剧情进展
- [ ] `pending-hooks.md` — 新伏笔埋设 + 伏笔回收 + 状态更新

**条件更新**：
- [ ] 是否有新角色出场？→ 按 `character-profile.md` 模板创建完整档案
- [ ] 是否有角色状态重大变化？→ 更新 `characters.md` 当前状态
- [ ] 是否有新系统/金手指/神器灵体出现？→ 按系统/金手指档案模板创建记录
- [ ] 是否有系统升级/功能解锁？→ 更新系统等级和成长记录，同步 `power-system.md`
- [ ] 是否有新地点/势力/规则？→ 更新 `world-setting.md`
- [ ] 是否有力量体系变化？→ 更新 `power-system.md`
- [ ] 是否有情感转折？→ 更新 `emotional-arcs.md`
- [ ] 是否跨5章或时间推进？→ 更新 `timeline.md`（含冲突检测）
- [ ] 是否跨幕或支线变化？→ 更新 `subplot-board.md`

### 2. 归档（必做）

- [ ] 将最终稿件从 `.novel-maker/temp/` 复制到 `novels/volume-XX/chapters/chXXX.md`
- [ ] 清理 `temp/` 中的 draft.md 和 revised.md（保留 audit.json）

### 3. 总结检查（必做）

- [ ] 当前章节是否为10的倍数？→ 生成小总结
- [ ] 当前章节是否为50的倍数？→ 生成大总结
- [ ] 当前章节是否为卷末？→ 生成卷总结

### 4. 规划偏离检查（每10章必做）

- [ ] 对比 `plan.md` 与实际剧情
- [ ] 若偏离，在【复盘结果摘要】中提示协调者

## 复盘师输出格式

```markdown
【复盘结果摘要】
- 章节：chXX
- truth-files 更新状态：已更新 X 个文件 [列出]
- 归档：novels/volume-XX/chapters/chXXX.md
- 临时文件清理：已完成
- 阶段总结触发：是/否（类型：小总结/大总结/卷总结）
- 规划偏离：无 / [偏离点描述]
- 章节摘要：50字以内
```

## 注意事项

1. 更新操作是增量的，不会覆盖已有内容
2. 如果某些文件不存在，使用对应模板创建初始版本
3. 所有强制检查项必须完成，否则不能在结果摘要中标记"已完成"
4. 每个 truth-file 的更新应参考对应的模板文件，确保结构完整
