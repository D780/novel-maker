# Chapter Complete Hook

## 概述

章节完成后的自动化更新流程。由复盘师 sub-agent 执行，协调者通过 Task 工具调度。

## 触发方式

由协调者调度复盘师 sub-agent 时触发。复盘师读取最终稿件后执行本 hook 的所有检查。

## 更新内容

### 1. 更新大纲

**文件**：`novels/outline.md`

**操作**：将章节状态从"待写"改为"已完成"，添加完成时间和摘要。

```markdown
## 第三幕：高潮
### 第50章 王都风云 [已完成] 2026-06-26
- 林风进入王都
- 遭遇皇室势力
- 古剑·风吟封印觉醒
- 字数：3200字
```

### 2. 更新记忆文件

**文件**：`truth-files/current-state.md`

**操作**：更新角色状态、世界状态、新获得的物品/技能。

### 3. 更新伏笔表

**文件**：`truth-files/pending-hooks.md`

**操作**：标记已回收的伏笔、添加新埋设的伏笔、更新状态。

### 4. 生成章节摘要

**文件**：`.novel-maker/summaries/chapter-XX-summary.md`

**内容**：章节主要事件、角色状态变化、新埋设伏笔、关键转折点。

## 强制检查清单（复盘师执行）

### 1. Truth-Files 更新（必做）

- [ ] `current-state.md` — 更新世界状态
- [ ] `pending-hooks.md` — 更新伏笔状态
- [ ] 是否有新角色/地点/势力？→ 更新 `characters.md` / `world-setting.md`
- [ ] 是否跨5章？→ 更新 `timeline.md` / `emotional-arcs.md` / `characters.md`
- [ ] 是否跨幕？→ 更新 `subplot-board.md` / `emotional-arcs.md`

### 2. 总结检查（必做）

- [ ] 当前章节是否为10的倍数？→ 必须生成小总结
- [ ] 当前章节是否为50的倍数？→ 必须生成大总结
- [ ] 当前章节是否为卷末？→ 必须生成卷总结

### 3. 规划偏离检查（每10章必做）

- [ ] 对比 `plan.md` 与实际剧情
- [ ] 若剧情偏离原计划，在【复盘结果摘要】中提示协调者

## 注意事项

1. 复盘师 sub-agent 执行此 hook，完成后返回【复盘结果摘要】给协调者
2. 如果某些文件不存在，跳过对应更新
3. 更新操作是增量的，不会覆盖已有内容
4. 所有强制检查项必须完成，否则不能在结果摘要中标记"已完成"

## 复盘师输出格式

复盘师完成本 hook 的所有检查后，返回以下摘要给协调者：

```markdown
【复盘结果摘要】
- 章节：chXX
- truth-files 更新状态：已更新 X 个文件 [列出]
- 临时文件清理：已完成
- 章节摘要：已生成 / 未触发
- 规划偏离：无 / [偏离点描述]
```
