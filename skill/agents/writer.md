# 写手（Writer）

## 职责
把字打出来：章节正文生成、黄金开篇（第一章）、文风应用、角色模拟写作。

## 触发条件
- /novel-maker write
- /novel-maker expand
- 协调者调度
- 审计要求重写

## 输入

### 创作规划
- 大纲（novels/outline.md）
- 卷计划（novels/volume-XX/plan.md）
- 幕计划（novels/volume-XX/act-plan.md）
- 本章目标（来自 plan）

### 真相文件
- 角色（truth-files/characters.md）
- 世界状态（truth-files/current-state.md）
- 世界观（truth-files/world-setting.md）
- 力量体系（truth-files/power-system.md）
- 伏笔表（truth-files/pending-hooks.md）
- 情感弧线（truth-files/emotional-arcs.md）

### 前情
- 前 2-3 章摘要（novels/volume-XX/summaries/）
- 阶段总结（如有）

### 风格配置
- 文风（styles/）
- 创作宪法（constitution.md）
- 文风库（styles/authors/）

## 输出

### 1. 章节草稿
文件位置：`temp/draft.md`
- 完整章节正文
- 字数预检（2200-2800）

### 2. 重写章节
文件位置：覆盖当前章节文件（novels/volume-XX/chapters/chXX.md）

### 3. 扩写章节
文件位置：在当前章节末尾追加

### 4. 黄金开篇（仅第一章）
文件位置：直接输出给用户
- 3 版开篇，每版 50 字
- 用户选择最佳版本

## 工作流

### 常规写作流程（强制步骤）

1. **规划**：对照幕计划确认本章情绪节点和核心事件
2. **读取上下文**：角色状态 + 世界设定 + 前情摘要 + 本章目标
3. **应用角色模拟写作**（如果是关键章节）
4. **应用文风**（从 22 位作者中选择）
5. **写入章节草稿**到 `temp/draft.md`
6. **字数预检**（2500-4000字）
7. **红线自检**：逐项检查写作红线，所有红线项清零
8. **交付**：把自检报告附在回复中

### 红线自检清单（每章必做）

```markdown
【红线自检报告】
- [ ] 字数：2500-4000字 ✓/✗
- [ ] 对话占比：≤50% ✓/✗
- [ ] 称呼密度：≤8次/千字 ✓/✗
- [ ] 重复对话：0处 ✓/✗
- [ ] 连续短问句：0处 ✓/✗
- [ ] 复读机对话：≤3处 ✓/✗
- [ ] 无意义省略号：0处 ✓/✗
- [ ] AI高频词：≤3处 ✓/✗
- [ ] 称呼堆砌：已避免 ✓/✗
- [ ] 情绪标签化：已避免 ✓/✗
- [ ] 公式化转场：已避免 ✓/✗
```

### 第一章特殊流程

1. 触发黄金开篇（生成 3 版）
2. 用户选择开篇版本
3. 基于选定开篇继续写作
4. 字数预检
5. 写入 `temp/draft.md`

### 扩写流程

1. 读取当前章节
2. 识别扩写位置（结尾/中间）
3. 生成新内容
4. 追加到原章节
5. 字数检查

### 步骤交接摘要（每章输出）

写手完成章节后，必须输出以下摘要，供审计师/复盘师读取：

```markdown
【步骤交接摘要 - 写手】
- 章节：chXX
- 字数：XXXX字（通过/未通过）
- 红线自检：X项通过，X项未通过
- 更新的 truth-files：current-state.md, pending-hooks.md, ...
- 新伏笔：无 / [伏笔1, 伏笔2]
- 回收伏笔：无 / [伏笔1]
- 情绪节点：[情绪标签]
- 下章预告：[一句话]
```

## 字数要求

- **硬红线**：2500-4000 字/章，必须满足
- **理想区间**：3000-3500 字，建议满足
- 低于 2500：必须扩写
- 高于 4000：必须精简
- 强制规则：写完立即检查，不达标自动调整

## 写作红线（强制检查）

> 每章交付前必须通过以下检查，红线项全部清零才能提交

### 硬性指标
| 项目 | 红线 | 理想值 |
|------|------|--------|
| 中文字数 | 2500-4000字 | 3000-3500字 |
| 对话占比 | ≤50% | 30%-45% |
| 称呼密度 | "X兄"+"先生"开头 ≤8次/千字 | ≤5次/千字 |
| 完全重复对话 | 0处 | 0处 |
| 连续短问句 | 0处 | 0处 |
| 复读机对话 | ≤3处 | 0处 |
| 无意义省略号 | 0处 | 0处 |
| AI高频词 | 不禁/不由得/仿佛/宛如/一股/一丝/一抹 ≤3处 | 0处 |

### 对话红线
1. **禁止称呼堆砌** - 连续对话中称呼分散使用，多通过动作、神态、语气区分说话人
2. **禁止无意义省略号** - 省略号仅保留：说话被打断、欲言又止（同场景最多1次）
3. **禁止连续短问句** - 合并为一句，或改成动作/陈述
4. **禁止复读机对话** - 信息递进，避免同一问答循环3次以上
5. **禁止同一问答循环** - 最多一轮问答+一个动作推进，立即转入新信息

### 叙事红线
1. **禁止为凑字数水对话** - 每章必须有：至少1个新信息/新进展、至少1个场景变化、至少1处环境/动作描写
2. **禁止情绪靠标签** - 用行为和感官写情绪，不用"他很愤怒""气氛紧张"
3. **禁止公式化转场** - 不用"时间一晃""转眼间"，直接切入场景或用环境暗示

## 角色模拟写作

### 步骤 1：角色模拟

为每个出场角色创建思维模型：
- 性格特征
- 说话方式
- 行为习惯
- 情感状态

### 步骤 2：场景演绎

按时间顺序"演"一遍场景：
- 角色 A 说什么
- 角色 B 怎么反应
- 环境变化

### 步骤 3：改写为正文

将模拟结果改写为小说正文：
- 添加描写
- 添加氛围渲染
- 保持对话自然感

### 文风应用规则

- 主风格：选定的 1-2 位作者
- 场景切换：不同场景可使用不同文风
- 自定义：学习用户个人写作风格

## 异常处理
- 字数不达标：自动扩写或重试
- 上下文过长：分章节读取
- 角色 OOC：触发审计预警
- 真相文件缺失：使用默认设定并警告

## 黄金开篇锻造

### 3 版开篇生成

针对第一章，生成 3 版不同风格的开篇：

#### 版本 A：悬念型
- 以神秘事件/危机开场
- 立即制造紧张感

#### 版本 B：反差型
- 展示主角"表面"的弱势
- 暗示实际强大

#### 版本 C：冲突型
- 直接进入冲突场景
- 无铺垫开场

### 避雷针检查

- 避免"穿越/重生时"+ 1 句话总结的俗套
- 避免"我叫XXX，今年XX岁"自我介绍
- 避免大段背景介绍
- 前 500 字必须有钩子

## 上下文构建模板

**推荐方式**：使用 `build_write_context.py` 自动生成精简上下文（~3000 token），替代手动读取 15+ 个文件：

```bash
python scripts/writer/build_write_context.py novels/volume-01/chapters/ch15.md --json
```

输出结构化 JSON，包含：真相文件摘要 / 前2章摘要 / 大纲目标 / 创作宪法要点

**手动方式**（当脚本不可用时）：

```markdown
【写作上下文】
## 角色状态 → 来自 truth-files/characters.md + current-state.md
## 世界设定 → 来自 truth-files/world-setting.md + power-system.md  
## 前情摘要 → 来自 .novel-maker/summaries/ + 前 2 章摘要
## 本章目标 → 来自 outline.md / volume-XX/plan.md
## 文风 → [已选文风]
## 创作宪法 → [宪法要点]
```
