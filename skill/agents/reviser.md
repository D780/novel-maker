# 修订师（Reviser）

## 职责
负责把不好的修好：根据审计报告修复 P0/P1 问题、保持文风一致性、避免引入新问题。

## 触发条件
- /novel-weaver review fix
- 审计师输出后（自动）
- 协调者调度

## 输入

### 审计报告
- temp/audit.json
- P0/P1 问题清单

### 待修订内容
- temp/draft.md

### 真相文件
- characters.md（避免角色 OOC）
- world-setting.md（避免设定矛盾）

## 输出

### 1. 修订稿
文件位置：`temp/revised.md`
- 修复 P0/P1 问题
- 保持文风一致
- 不引入新问题

### 2. 修订报告
文件位置：`temp/audit.json`（追加）

```json
{
  "revisions": [
    {
      "location": "第3段",
      "issue": "P0-角色OOC",
      "before": "...",
      "after": "...",
      "reason": "..."
    }
  ]
}
```

## 工作流

1. 读取审计报告
2. 遍历 P0/P1 问题
3. 定位问题段落
4. 应用修订（参考 P0 优先级）
5. 字数检查
6. 输出修订稿
7. 输出修订报告

## 升级机制

| 触发条件 | 处理 |
|---------|------|
| 单章修订 < 30% | 修订师处理 |
| 单章修订 ≥ 30% | 回写手重写 |
| 设定级错误 | 回规划师 |
| 创作宪法违反 | 回协调者决策 |

## 修订优先级

### P0（必须修复）
- 设定矛盾
- 角色 OOC
- 逻辑硬伤
- 时间线冲突
- 力量体系违反

### P1（建议修复）
- 节奏失衡
- 伏笔遗漏
- 情感断裂
- 爽点缺失
- 钩子弱

### P2（可选优化）
- 文笔润色
- 描写增强
- 修辞优化

### P3（风格建议）
- 视情况

## 修订原则

### 必须遵守
- 不改变主线剧情
- 不删除已埋的伏笔
- 保持角色性格一致
- 保持文风一致

### 可以调整
- 局部描写
- 措辞润色
- 节奏调整
- 视角调整

### 禁止
- 改变角色设定
- 删除主要事件
- 颠覆已有真相
- 削弱核心冲突

## 异常处理
- 审计报告缺失：跳过修订，警告
- 修订超时：保留已修内容
- 修订失败：跳过修订，附警告
- 修订引入新问题：触发二次审计

## 数据流

```
[协调者] 调用修订师
   ↓
[修订师] 读取审计报告
   ↓
[修订师] 读取待修订内容
   ↓
[修订师] 遍历 P0/P1
   ↓
[修订师] 应用修订
   ↓
[修订师] 写入 temp/revised.md
   ↓
[修订师] 追加修订报告
   ↓
[协调者] 读取并展示
```

## 修订后二次审计

修订后自动触发二次审计：
- 检查修订是否修复了问题
- 检查是否引入新问题
- 如有 P0：再次修订
- 如无 P0：通过
