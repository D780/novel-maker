# 反AI表达规则

> 检测并修正AI写作中的"AI味"，让小说更像真人写的

## 7层检测体系

### 第1层：广告词过度（Ad Overuse）

**问题：** AI倾向于使用过多的修饰词和"广告词"

**检测模式：**
- 连续3个以上形容词修饰同一名词
- 使用"完美"、"极致"、"无与伦比"等绝对化词汇
- 使用"仿佛"、"宛如"、"恰似"等比喻词过度（每章超过3次）

**修正建议：**
- 删除冗余形容词，保留最有力的1-2个
- 用具体描写替代抽象形容
- 减少比喻，增加直接描写

### 第2层：情感标签化（Emotion Tagging）

**问题：** AI倾向于直接告诉读者角色的情感，而非展示

**检测模式：**
- "他感到愤怒"、"她心中一喜"等直接情感陈述
- "气氛变得紧张"等氛围标签
- "一股暖流涌上心头"等陈词滥调

**修正建议：**
- 用动作展示情感（"他猛地拍了一下桌子"）
- 用环境烘托氛围（"房间里安静得能听到钟表的滴答声"）
- 用对话暗示情感

### 第3层：对话形式化（Dialogue Formality）

**问题：** AI生成的对话往往过于正式、书面化

**检测模式：**
- 对话中使用"您"过多（除非角色设定需要）
- 对话句子过长（超过30字）
- 对话中使用书面语（"然而"、"因此"、"尽管如此"）
- 对话缺乏口语化表达（嗯、啊、呢、吧、嘛）

**修正建议：**
- 增加口语化表达
- 缩短对话句子
- 添加对话中的打断、犹豫、重复

### 第4层：结构闭合化（Structural Closure）

**问题：** AI倾向于在每段/每章结尾做总结

**检测模式：**
- 段落结尾用"总之"、"总而言之"、"由此可见"
- 章节结尾用"这一切，只是开始"等套路
- 每个场景结束后都有"反思段"

**修正建议：**
- 删除总结性语句
- 用悬念或行动结尾
- 留白，让读者自己体会

### 第5层：转场公式化（Transition Formula）

**问题：** AI的场景转场往往生硬或套路

**检测模式：**
- "时间一晃，来到了..."、"转眼间..."
- "与此同时，在另一个地方..."
- "另一边，XXX也在..."

**修正建议：**
- 直接切入新场景
- 用环境变化暗示时间流逝
- 用角色行动自然转场

### 第6层：总结倾向（Summary Tendency）

**问题：** AI倾向于总结而非叙述

**检测模式：**
- "接下来的几天里..."、"在接下来的日子里..."
- "经过一番努力..."、"经过长时间的..."
- "终于，在XXX的帮助下..."

**修正建议：**
- 展示具体过程，而非总结结果
- 选取关键场景详细描写
- 用时间跳跃替代总结

### 第7层：信息曝光（Info Exposure）

**问题：** AI倾向于过早或过多地暴露信息

**检测模式：**
- 角色突然知道不该知道的信息
- 旁白透露过多背景信息
- 对话中塞入过多设定

**修正建议：**
- 控制信息释放节奏
- 用角色视角限制信息
- 通过事件自然揭示信息

## 常见AI-slop模式（60+）

### 广告词类
- 完美无瑕、无与伦比、登峰造极
- 独一无二、前所未有、史无前例
- 惊天动地、惊世骇俗、骇人听闻
- 气势磅礴、波澜壮阔、恢弘大气
- 精妙绝伦、巧夺天工、鬼斧神工

### 情感标签类
- 心中一喜、心中一怒、心中一惊
- 感到一阵温暖、感到一阵心酸
- 气氛变得紧张、气氛变得凝重
- 一股暖流涌上心头、一阵寒意袭来
- 不由得、不禁、忍不住

### 对话套路类
- "你..."他欲言又止。
- "哼！"他冷哼一声。
- "呵..."他轻笑一声。
- "你！"他怒目而视。
- "哦？"他挑了挑眉。

### 结构套路类
- 这一切，只是开始。
- 他知道，真正的挑战还在后面。
- 命运的齿轮，开始转动。
- 故事，才刚刚开始。
- 这一战，注定载入史册。

### 转场套路类
- 时间一晃，来到了三天后。
- 与此同时，在另一个地方...
- 转眼间，一个月过去了。
- 就这样，又过了几天。
- 接下来的日子里...

### 描写套路类
- 他的眼神变得深邃起来。
- 他的嘴角微微上扬。
- 他的眼中闪过一丝精光。
- 他的气势瞬间暴涨。
- 他的声音充满了威严。

### 总结套路类
- 总之，这一切都结束了。
- 由此可见，他的实力不容小觑。
- 毫无疑问，这是一场胜利。
- 无论如何，他都不会放弃。
- 毫无疑问，他是一个强者。

## 使用方式

### 写作时提醒

在写作时，对照此规则避免AI味：

```
根据 anti-ai-expressions.md 的规则，避免使用广告词和情感标签
```

### 审查时检查

在审查章节时，对照此规则检测AI味：

```
/novel-maker review 检查第5章的AI味
```

### 自查清单

写作完成后，检查以下项目：
- [ ] 是否有连续3个以上形容词？
- [ ] 是否有直接情感陈述？
- [ ] 对话是否过于正式？
- [ ] 是否有总结性语句？
- [ ] 是否有公式化转场？
- [ ] 是否有信息泄露？
